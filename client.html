Jenkins API Client {Client}

Client libraries for communicating with a Jenkins CI server and programatically managing jobs.

  • Authentication

About this Class

This class acts as the bridge between the other Jenkins interface specific classes and the Jenkins server. All GET and POST requests are handled by this class. This class also performs some error handling for the requests received from the server. This class also creates objects to other classes on the fly.

Authentication

This project supports two types of password-based authentication. You can just you the plain password by using password parameter. If you don't prefer leaving plain passwords in the credentials file, you can encode your password in base64 format and use password_base64 parameter to specify the password either in the arguments or in the credentials file.

Initializing the Client

As discussed earlier, you can either specify all the credentials and server information as parameters to the Client or have a credentials file and just parse the yaml file and pass it in. The following call just passes the information as parameters.

@client = JenkinsApi::Client.new(:server_ip => '0.0.0.0',
          :username => 'somename', :password => 'secret password')
# The following call will return all jobs matching 'Testjob'
puts @client.job.list("^Testjob")

The following example passes the YAML file contents. An example yaml file is located in config/login.yml.example.

@client = JenkinsApi::Client.new(YAML.Load_file(File.expand_path('~/.jenkins_api_client/login.yml', __FILE__)))
# The following call lists all jobs
puts @client.job.list_all

Accessing other Classes

This is the main class under the JenkinsApi namespace and all other classes are subclasses to this Client class. Once you create an object to the Client class, you can easily access other classes by just calling the methods matching the class name. For example, to call a method in the job class just call the method job using your client object and then the intended method you wanted to call on the job class. The job method on the client class creates an object to the job class by passing all required credentials.
